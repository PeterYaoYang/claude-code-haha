# 源码阅读顺序

## 新手推荐顺序

### 1. `src/entrypoints/cli.tsx`

看什么：

- 启动怎么分流
- 什么是 fast path

### 2. `src/main.tsx`

看什么：

- runtime 是怎么装配起来的
- setup、commands、agents、context 是怎么串起来的

### 3. `src/commands.ts`

看什么：

- 用户入口怎么组织
- command 和 tool 的区别

### 4. `src/tools.ts`

看什么：

- 模型可调用的能力是怎么注册的

### 5. `src/constants/tools.ts`

看什么：

- agent 权限边界
- 为什么不同 agent 工具权限不同

## 读源码时要记住

不要一开始逐行啃。

优先问：

- 这个文件解决什么问题
- 它在整套 agent 系统里扮演什么角色
- 它和 model、agent、runtime、tool 哪一层相关
